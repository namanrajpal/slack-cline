How sline works under the hood

Conceptually, sline is a small brain that turns Slack conversations into structured tasks, sends them to a Cline runner, and mirrors the results back into Slack and your git provider.

Step 1

Capture context

sline captures the Slack thread, links, and snippets, turning them into a structured task spec for Cline.

Step 2

Call the runner

The spec is sent to your Cline runner (via API/WebSocket), along with repo, branch, and constraints.

Step 3

Cline edits code

Cline modifies files, runs tests, and pushes a feature branch. sline opens or updates the PR and tracks status.

Step 4

Review & refine

Engineers review in git, discuss in Slack, and tag sline again to apply fixes on the same branch.

What is sline, exactly?

Is this a Slack app, or a Cline wrapper?

Both. sline is a Slack app plus a small orchestration layer that talks to a dedicated Cline runner service you control. It doesn’t replace your agent — it gives you a way to steer it from Slack.

Which git providers does it work with?

The concept works with any git provider that supports branches and PRs (GitLab, GitHub, etc.). The first integrations will likely focus on GitLab-style APIs.

Where does the runner live?

You run the Cline runner in your own infra (e.g., AWS Fargate/EC2/K8s). sline just sends structured tasks and receives status updates.
